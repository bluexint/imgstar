## 1. 报告目标
本报告用于冻结当前共识，避免架构讨论跨层混杂，作为后续架构设计与实现评审的唯一基线。

## 2. 范围与边界（L0）
- 产品目标：降低无效回源成本优先。
- 用户画像：普通用户优先（只填 Key 可用），高级配置可选开启。
- MVP 目标：功能可用优先，不以成本指标达标作为上线门槛。
- 责任边界：平台不内置业务鉴权，主做防穿透、可观测、可扩展。

## 3. 能力分层（L1）
- 上传能力：上传编排、编号生成、失败处理。
- 预览能力：图片预览（按屏幕像素缩略图）、hash 校验（默认开启）、本地缓存管理；流媒体/水印/HLS 等能力通过 Hook 扩展。
- 插件能力：预处理 Hook、权限控制、插件扩展点。
- 设置能力：基础设置与高级开关。
- 防护能力：WAF 规则、路径约束、范围限制。
- 可观测能力：请求级追踪、日志筛选。
- 国际化能力：多语言包、键值治理、回退链。
- 主题能力：低饱和科技蓝视觉系统、深浅模式一致。

## 4. 关键流程（L2）
- 上传主流程：选择文件 -> 预处理 Hook（串行）-> 编号预占 -> 上传 -> 结果记录。
- 异常流程：任一 Hook 抛错即停止当前上传，返回明确错误信息。
- 预览流程：图片按屏幕像素生成缩略图 -> 本地缓存 -> hash 校验；映射缺失不阻塞主流程。
- 防护流程：WAF 正则白名单判定 -> Challenge -> Block。
- 排障流程：按 traceId 聚合查看上传链路日志。
- 语言流程：启动加载目标语言包，缺失键按回退链兜底。

## 5. 契约冻结（L3）

### 5.1 对象 Key 语法契约
- 结构：类型前缀 / 稳定命名空间 / 顺序编码.扩展名。
- 示例：img/public/000000123.jpg。
- 约束：
  - 编号位数可配置，默认 9 位。
-  - 采用编号池状态机：`free -> reserved -> active -> deleted -> cooling -> free`。
-  - 删除后的编号必须复用，但仅在冷却期结束后复用（避免 CDN 旧缓存与新对象混淆）。
-  - 采用两阶段发布：`reserved`（预占）-> `active`（可访问）。
-  - 仅 `active` 进入外部可见范围；非 `active` 一律在边缘层拦截。
-  - 禁止使用时间戳作为公开路径分片依据（避免可枚举绕过面）。

### 5.2 Hook 契约
- 执行顺序：串行（按 priority 排序）。
- 失败语义：抛错即停，终止当前流程。
- 错误返回：统一结构，便于日志与界面展示。
- Hook 类型：按模块分发（UploadHook / PreviewHook / SecurityHook / LogHook 等）。
- 注册契约：统一注册 API，插件声明 hook_type + hook_fn + priority。

### 5.3 插件权限契约
- 默认权限：不可联网。
- 放开方式：显式授权。
- Rust 外部进程能力：归插件范畴，不纳入 MVP。

### 5.4 可观测契约
- 必须字段：time、module、level、traceId、action、result。
- 追踪粒度：请求级 traceId 必须具备。
- 日志保留：由开发者配置，系统提供选项。

### 5.5 i18n 契约
- 定位：基础能力。
- 实现：半静态（构建期预编译 + 启动加载 + 运行期轻量取值）。
- 首批语言：中文简体、英文。
- 键值组织：按模块命名空间。
- 回退链：用户语言 -> 中文简体 -> 英文。

## 6. 策略层（L4）

### 6.1 已冻结策略
- 防护主轴：WAF 正则白名单 + 命名规范 + 范围限制。
- 厂商策略：每家厂商独立适配器与 Hook，不做统一能力并集。

- 映射表策略：默认关闭，仅在维护性需要时启用，不参与主键生成。
- 视觉策略：低饱和科技蓝（偏冰岛蓝），保证层次一致性。

## 9. 架构草案（第一轮对齐）

### 9.1 模块清单与职责
- AppShell（Astro + Vue3）：页面路由、全局状态挂载、主题与语言初始化。
- PreviewModule：预览框架与 Hook 扩展中心，MVP 支持图片预览与 hash 校验，流媒体/水印/HLS 等能力通过 Hook 扩展。
- UploadOrchestrator：上传编排中枢，负责 traceId 生成、Hook 串行执行、失败中止与状态回传。
- KeyAllocator：编号预占、游标持久化（heed）、编号位数校验、不可复用约束。
- StorageAdapterRuntime：按用户选择加载厂商适配器插件，统一调用上传/查询/删除能力。
- PluginRuntime：Hook 统一注册与执行中心，支持 Rust 动态库插件（libloading 加载），函数指针直接调用。统一注册契约，按 hook_type 分发到各模块专属 Hook（UploadHook/PreviewHook/SecurityHook 等），模块自治。
- SecurityPolicyCenter：WAF 规则模板、本地路径语法校验、策略更新入口。
- NameMappingStore：对象 Key 与显示名本地映射（heed）、导入导出、查询索引。
- LogCenter：请求级 traceId 日志收集、筛选、导出与滚动清理。
- I18nCenter：语言包加载、命名空间路由、缺失键回退链。
- ThemeCenter：语义 Token 注入、深浅模式切换、组件层级一致性控制。

### 9.2 模块输入输出（边界契约）
- UploadOrchestrator 输入：文件列表、上传目标配置、插件链配置。
- UploadOrchestrator 输出：任务状态流（queued/running/success/failed）、结构化错误对象。
- KeyAllocator 输入：文件类型、时间分片开关、编号位数配置。
- KeyAllocator 输出：符合语法契约的唯一对象 Key。
- StorageAdapterRuntime 输入：对象 Key、文件流、元信息。
- StorageAdapterRuntime 输出：厂商响应的标准化结果对象。
- PluginRuntime 输入：Hook 上下文（文件、Key、配置、traceId）。
- PluginRuntime 输出：变更后的上下文或错误终止信号。
- PreviewModule 输入：对象 Key / 文件数据、预览配置（缩略图尺寸、是否启用 hash 校验）。
- PreviewModule 输出：预览数据流、本地缓存状态、hash 校验结果。
- LogCenter 输入：全链路事件。
- LogCenter 输出：筛选结果、导出文件、聚合统计。

### 9.3 依赖方向与禁止耦合
- 允许依赖：AppShell -> UploadOrchestrator/PluginRuntime/LogCenter/I18nCenter/ThemeCenter/PreviewModule。
- 允许依赖：UploadOrchestrator -> KeyAllocator/PluginRuntime/StorageAdapterRuntime/LogCenter。
- 允许依赖：PreviewModule -> PluginRuntime（Hook 调用）、LogCenter。
- 允许依赖：PluginRuntime -> LogCenter（仅日志写入）。
- 禁止依赖：StorageAdapterRuntime 直接调用 UI 层。
- 禁止依赖：PluginRuntime 直接读写设置持久层（必须经 Settings API）。
- 禁止依赖：NameMappingStore 参与安全决策（仅用于可读性与运维辅助）。

### 9.4 关键异常语义（冻结）
- Hook 异常：立即停止当前上传任务，记录 traceId 与错误码。
- 超时语义：超时视为独立错误类型（不是普通业务错误），由运行时监控器判定并中止当前阶段。
- 长任务语义：对视频预处理等长耗时模块，优先按“进度心跳超时”判定，而不是只看固定时长超时。
- Hook 文件处理约定：修改文件内容则返回新 buffer（由 Orchestrator 覆盖）；仅添加元数据注解则返回 metadata（高级模式处理）。
- 预占号成功但上传失败：编号回收至 `free`（可立即复用）。
- 删除对象后编号处理：先标记 `deleted`，进入 `cooling` 后再回收，冷却时长 >= CDN TTL。
- 适配器失败：按统一错误对象回传，不吞错。
- i18n 缺失键：按回退链降级，禁止渲染空白文案。

## 10. 第二轮对齐：上传链路事件模型（L3 契约补充）

### 10.1 上传链路事件集合（定义阶段）
上传任务从"选文件"到"结果记录"会产生以下关键事件，所有事件必须共享 traceId：

| 事件名 | 触发时机 | 产物流向 |
|--------|---------|---------|
| `upload:task_created` | 用户确认上传 | LogCenter + PluginRuntime |
| `upload:key_allocated` | 编号预占成功 | LogCenter + NameMappingStore |
| `upload:hook_before_process` | Hook 执行前（每个 Hook 处触发） | LogCenter + PluginRuntime |
| `upload:hook_after_process` | Hook 执行后 | LogCenter |
| `upload:hook_error` | Hook 执行失败 | LogCenter + UI（错误通知） |
| `upload:adapter_start` | 开始调用存储适配器 | LogCenter |
| `upload:adapter_success` | 适配器返回成功 | LogCenter + NameMappingStore |
| `upload:adapter_error` | 适配器返回错误 | LogCenter + UI（错误通知） |
| `upload:progress_report` | 模块上报阶段进度 | LogCenter + UI（进度条） |
| `upload:stage_timeout` | 任一阶段超时 | LogCenter + UI（错误通知） |
| `upload:stage_slow` | 任一阶段超过慢阈值但未超时 | LogCenter |
| `upload:task_success` | 整个链路完成 | LogCenter + NameMappingStore |
| `upload:task_failed` | 链路中止 | LogCenter + UI（错误通知） |

### 10.2 事件对象结构（冻结字段）
所有事件遵循此基础结构：
```json
{
  "traceId": "string（UUID）",
  "timestamp": "ISO8601",
  "module": "upload",
  "eventName": "string",
  "status": "success | failed | skipped",
  "errorCode": "string | null（仅在失败时填充）",
  "errorMessage": "string | null（用户可读，支持 i18n）",
  "stack": "string | null（仅失败事件记录堆栈）",
  "duration": "number（毫秒）",
  "context": {}
}
```

### 10.4 事件与 Hook 上下文的绑定（约束）
- PluginRuntime 调用 Hook 时，必须在 context 中传递当前 traceId。
- Hook 内部日志输出由插件开发者负责（如需记录，自由调用 Logger）。
- Hook 返回的任何错误必须携带 errorCode 字段，便于 PluginRuntime 捕获并记录。
- Hook 如果修改文件内容，返回新 buffer；如仅添加元数据，返回 metadata 对象（高级模式处理）。
- 长任务 Hook（如视频预处理）应周期性上报 `upload:progress_report`。
- 如果进度长时间不变（例如停在 50% 且心跳超时），运行时判定为卡死并触发 `upload:stage_timeout`。

### 10.5 事件与 i18n 的绑定
- errorMessage 必须对应一个 i18n 键，不允许硬编码文案。
- i18n 键命名规范：`upload.errors.{errorCode}` 或 `upload.infos.{eventName}`。
- 缺失键时按语言回退链处理，最终显示为"未知错误"。

### 10.6 事件与 LogCenter 的约定
- LogCenter 收到任何事件都必须持久化（文件或本地数据库）。
- 筛选维度至少支持：module、eventName、status、errorCode、traceId、时间范围。
- 导出格式可选 JSON 或 CSV。
- 日志保留周期与清理方式由开发者自定义（按大小滚动 / 按天数清理 / 手动清理）。
- 必须支持阶段耗时统计：每个阶段记录开始时间、结束时间、耗时分布（P50/P95）。
- 必须支持超时监控：超过 timeoutMs 触发 `upload:stage_timeout` 并中止阶段。
- 必须支持慢调用监控：超过 slowThresholdMs 触发 `upload:stage_slow` 但不立即中止。
- 必须支持进度心跳监控：连续超过 heartbeatTimeoutMs 未收到新进度上报，触发超时判定。
- 进度卡死判定规则：进度值与 seq 在 heartbeatTimeoutMs 窗口内均无变化，视为卡死。
- stack 记录规则：仅 `status=failed` 或 `eventName in (*_error, *_timeout)` 时记录 stack。

### 10.7 待定参数
- 【已拍板】errorCode 最小集合（见 10.8）。
- 【已拍板】stack 仅失败事件记录，模式固定 compact。
- 【已拍板】requestHeaders debug 仅白名单字段输出，敏感字段强制掩码。
- 【已拍板】config 快照默认关闭（高级设置可开启）。

- 【已拍板】各阶段保守默认值（见 11.5 与 10.9）。

### 10.9 跳号与缓存穿透治理（已拍板）

问题：顺序编号场景中，跳号与可枚举分片（如时间戳）会放大未命中面，增加缓存穿透压力。

策略：
- 编号池状态机：`free -> reserved -> active -> deleted -> cooling -> free`。
- 删除后必须复用：编号进入 `cooling`，冷却结束后回到 `free`，防止长期跳号。
- 仅 `active` 编号允许访问；`reserved/deleted/cooling` 在边缘层直接拦截。
- 冷却时长规则：`reuseDelayMs >= max(cdnTTL, edgeNegativeCacheTTL)`。
- WAF 更新时机：上传队列完成后批量更新（而非每文件更新），降低规则抖动。
- 路径策略：禁用公开时间分片，固定命名空间白名单（如 `img/public/*`），避免通过年份/月份枚举绕过。

结果：
- 保留顺序编号与正则策略。
- 删除可复用，不要求把全部编号“永久占满”。
- 显著降低“跳号 + 时间枚举”导致的可探测空洞面。

## 11. 第三轮对齐（最佳实践默认方案）

### 11.1 取消语义（默认冻结）
- 新增事件：`upload:task_cancelled`（用户主动取消）。
- 取消不是失败：在统计中单列 `cancelled`，不并入 `failed`。
- 取消优先级：用户取消 > 慢调用告警 > 超时判定。
- 取消传播：Orchestrator 广播取消信号到当前阶段，Hook 与 Adapter 必须实现可中断检查点。

默认字段（context）：
```json
{
  "cancelBy": "user | system",
  "cancelAtStage": "hook_n | key_allocation | adapter",
  "elapsedMs": "number",
  "reason": "string | null"
}
```

### 11.2 重试与幂等矩阵（默认冻结）
- 幂等操作默认允许自动重试（指数退避）。
- 非幂等操作默认不自动重试，仅返回错误并提示人工重试。
- 推荐退避：`baseDelayMs = 300`，`maxRetries = 2`（MVP 保守默认）。

默认矩阵：

| 操作 | 是否幂等 | 自动重试 |
|------|----------|----------|
| key_allocation | 是（本地事务） | 否（失败立即返回） |
| hook_process | 视插件声明 | 否（默认禁重试） |
| adapter_putObject | 视对象键是否唯一且可覆盖策略 | 是（网络错误时） |
| adapter_multipart_part_upload | 是（按 partNumber） | 是 |
| adapter_complete_multipart | 否（有副作用） | 否 |

插件声明约束：
- 插件在 manifest 必须声明 `idempotent: true|false`，默认 `false`。

### 11.3 脱敏规则（默认冻结）
- stack 默认记录关键层级，不记录完整本地绝对路径。
- requestHeaders 默认白名单输出，其余字段一律掩码。
- debug 模式可扩展字段，但必须二次确认并显示风险提示。

默认敏感字段掩码清单：
- `authorization`
- `x-amz-security-token`
- `x-amz-signature`
- `cookie`
- `set-cookie`
- `accessKey`
- `secretKey`

默认白名单头：
- `content-type`
- `content-length`
- `cache-control`
- `x-request-id`
- `x-amz-request-id`

### 11.4 本地数据版本与迁移（默认冻结）
- NameMappingStore 与 LogCenter 均引入 `schemaVersion`。
- 启动时执行迁移检查：版本不一致则按迁移脚本升级。
- 迁移必须可回滚：迁移前先做快照备份。
- 迁移过程写入事件：`storage:migration_started`、`storage:migration_finished`、`storage:migration_failed`。

默认元数据字段：
```json

  "schemaVersion": "string",
  "createdAt": "ISO8601",
  "updatedAt": "ISO8601",
  "migratedFrom": "string | null"
```

### 11.5 建议默认值（可覆盖）
- 所有默认值由统一配置中心注入，禁止在业务模块中硬编码。

  - `upload.progress.report_interval_ms = 1000`

  - `log.stack.mode = compact`（仅失败事件记录）
  - `log.snapshot.enabled = false`
  - `waf.update.trigger = queue_completed`
  - `upload.reuse.delay_ms = 900000`

高级设置新增项：
- `编号复用冷却(ms)` -> 对应 `reuseDelayMs`。

## 12. 第四轮对齐：插件权限模型（默认冻结）

### 12.1 设计目标
- 默认最小权限：插件安装后仅有本地计算能力，不具备网络访问与高风险系统能力。
- 权限可见可控：每个权限必须可解释、可授权、可撤销、可审计。
- 失败优先安全：未声明权限或权限校验失败时，默认拒绝执行对应能力。

### 12.2 权限分级
- `safe`：低风险权限，默认允许。
- `sensitive`：中高风险权限，需要显式授权。
- `restricted`：高风险权限，需二次确认并支持按插件单独开关。

默认权限分级表：

| 权限键 | 说明 | 级别 | 默认 |
|--------|------|------|------|
| `hook.process` | 执行 Hook 文件处理 | safe | 允许 |
| `storage.read` | 读取对象元信息（通过受控 API） | safe | 允许 |
| `storage.write` | 写入对象（通过受控 API） | sensitive | 拒绝 |
| `network.outbound` | 访问外网 | restricted | 拒绝 |
| `process.spawn` | 启动本地外部进程 | restricted | 拒绝 |
| `fs.read` | 读取工作目录外文件 | sensitive | 拒绝 |
| `fs.write` | 写入工作目录外文件 | restricted | 拒绝 |
| `env.read` | 读取环境变量 | sensitive | 拒绝 |

### 12.3 插件 manifest 契约（冻结）
插件必须携带权限声明，未声明权限视为无权限：

```json
{
  "id": "string",
  "name": "string",
  "version": "string",
  "apiVersion": "string",
  "idempotent": false,
  "permissions": {
    "hook.process": true,
    "storage.read": true,
    "storage.write": false,
    "network.outbound": false,
    "process.spawn": false,
    "fs.read": false,
    "fs.write": false,
    "env.read": false
  }
}
```

校验规则：
- 缺失 `apiVersion` 或 `permissions` 时拒绝加载插件。
- 声明了未识别权限键时拒绝加载（防止权限拼写绕过）。
- `restricted` 级权限即使声明为 true，未授权前仍按 false 执行。

### 12.4 授权流程（默认冻结）
- 首次安装：展示权限清单与风险等级，不默认勾选敏感权限。
- 首次调用受限能力：运行时二次弹窗确认（按插件、按权限）。
- 授权粒度：插件级 + 权限级，支持随时撤销。
- 撤销行为：撤销后正在运行任务不中断，下次调用立即生效。

### 12.5 拒绝策略（默认冻结）
- 未授权调用：返回 `PERMISSION_DENIED`，并记录拒绝事件。
- 权限升级请求：返回 `PERMISSION_ESCALATION_REQUIRED`，不自动放行。
- 鉴于当前主路径采用签名策略，风险评分不作为 MVP 阻断条件。

### 12.6 权限审计日志（冻结字段）
新增事件：
- `plugin:loaded`
- `plugin:permission_prompted`
- `plugin:permission_granted`
- `plugin:permission_denied`
- `plugin:permission_revoked`
- `plugin:permission_violation`

审计事件基础结构：
```json
{
  "traceId": "string | null",
  "timestamp": "ISO8601",
  "pluginId": "string",
  "pluginVersion": "string",
  "eventName": "string",
  "permission": "string",
  "decision": "granted | denied | revoked",
  "reason": "string | null",
  "operator": "user | system",
  "stack": "string | null"
}
```

### 12.7 与现有模型的对齐关系
- 与 10 章对齐：权限拒绝属于错误事件，纳入同一 traceId 追踪链。
- 与 11 章对齐：重试逻辑不绕过权限校验，重试前必须再次检查权限状态。
- 与 9 章对齐：PluginRuntime 是权限唯一执行入口，其他模块不得直接放行插件调用。

### 12.8 待定参数
- 【已拍板】权限弹窗支持“本次允许/始终允许”双模式。
- 【已拍板】风险阈值不进入 MVP 阻断。
- 【已拍板】`restricted` 权限不要求管理员口令（二次确认保留为可选开关）。
- 【待定】权限审计日志默认保留周期。

## 15. 目录设计（结合 UI + 依赖 + Hook 解耦）

### 15.1 目录设计目标
- 目标 1：按“领域能力”分层，而不是按页面临时堆文件。

- 目标 3：后续可平滑接入 Tauri Rust 侧能力，不破坏前端结构。

### 15.3 分层依赖规则（强约束）
- `apps/desktop` 可以依赖 `packages/*`，反向依赖禁止。
- `packages/domain` 不允许依赖 UI 框架。
- `packages/contracts` 不依赖其他业务包，必须保持最薄。
- `packages/adapters/*` 仅依赖 `contracts` 与最小 runtime API。
- `packages/hooks/*` 不允许直接访问持久层，必须经 runtime 提供的受控上下文。

### 15.4 与你当前模块的映射
- 上传页面：`apps/desktop/src/pages/upload` + `packages/domain/src/upload`。
- 预览页面：`apps/desktop/src/pages/preview` + `packages/domain/src/preview`。
- 插件页面：`apps/desktop/src/pages/plugins` + `packages/runtime/src/plugin-runtime`。
- 设置页面：`apps/desktop/src/pages/settings` + `packages/domain/src/settings`。
- 开发者日志页：`apps/desktop/src/pages/devtools` + `packages/domain/src/logging` + `packages/storage/src/log-store`。

### 15.5 Hook 解耦后的目录落点
- Hook 接口定义：`packages/contracts/src/hook.ts`。
- Hook 调度器：`packages/runtime/src/plugin-runtime/hook-runner.ts`。
- Hook 上下文工厂：`packages/runtime/src/plugin-runtime/hook-context.ts`。
- 内置 Hook：`packages/hooks/*`。
- 第三方 Hook：建议放 `plugins-local/`（本地目录，运行时扫描）。

### 15.6 Tauri 与前端边界（防止耦合）
- 前端只调用 `src-tauri/commands` 暴露的能力，不直接关心 Rust 内部模块。
- Rust 侧外部进程（如 ffmpeg）未来放到 `src-tauri/src/infra/process/`。
- 凭证与敏感信息只存 Rust 安全存储层，前端仅拿引用 ID。

### 15.7 落地迁移清单（建议顺序）
1. 先建 `packages/contracts`，把事件/Hook/适配器接口冻结。
2. 再建 `packages/domain` 与 `packages/runtime`，跑通上传主链路。
3. 然后接入 `packages/adapters/cloudflare-r2`，只打通 MVP 厂商。
4. 最后迁移 UI 页面到 `apps/desktop/src/pages/*`，接 runtime。

### 15.9 目录命名规范与 import 别名（冻结）

目录命名规范：
- 包目录统一使用 `kebab-case`，例如 `plugin-runtime`、`name-mapping-store`。
- 代码文件命名：
  - 领域服务：`*.service.ts`
  - 运行时协调器：`*.runtime.ts`
  - 数据模型：`*.model.ts`
  - 契约与类型：`*.types.ts`
  - 错误定义：`*.errors.ts`
  - 常量：`*.constants.ts`
- 页面组件：`PascalCase.vue`，组合式函数：`use*.ts`。
- 测试文件：与被测文件同目录，后缀 `.spec.ts`。

跨包 import 规则：
- 禁止跨层相对路径穿透（如 `../../../../domain/...`）。
- 强制使用别名导入，禁止深层私有文件直连。
- 仅允许从包的 `index.ts` 导出面进入。

建议别名映射：
- `@app/*` -> `apps/desktop/src/*`
- `@domain/*` -> `packages/domain/src/*`
- `@runtime/*` -> `packages/runtime/src/*`
- `@contracts/*` -> `packages/contracts/src/*`
- `@adapters/*` -> `packages/adapters/*/src/*`
- `@storage/*` -> `packages/storage/src/*`

边界检查建议：
- 使用 lint 规则限制非法 import（按层限制）。
- CI 增加“架构边界检查”步骤，发现跨层依赖直接失败。

### 15.10 统一配置管理规范（冻结）

配置来源优先级（高 -> 低）：
1. 运行时临时覆盖（仅当前会话）
2. 高级设置持久化配置（本地）
3. 环境级配置文件（如 `config/default.toml`）
4. 系统内置默认值

配置读取规则：
- 业务代码只能通过 `ConfigCenter` 读取，不允许直接访问原始配置文件。
- 所有配置键采用 `domain.group.key` 规范。
- 读取时必须带类型校验与范围校验（超出范围回退默认值并记录告警）。

建议配置键分组：
- `upload.timeout.*`
- `upload.progress.*`
- `upload.reuse.*`
- `waf.update.*`
- `waf.rule.*`
- `log.stack.*`
- `log.retention.*`
- `plugin.permission.*`

热更新边界：
- 可热更新：日志级别、慢阈值、心跳间隔、WAF 更新策略。
- 需重启生效：编号位数上限、存储后端切换、插件权限基线。

## 17. P0 参数建议拍板（草案）

说明：本节为“先定一版可执行默认值”，用于直接进入实现，后续可通过高级设置覆盖。

### 17.1 错误码分层归属（建议）
- domain 层：仅抛业务语义码（如 `UPLOAD_VALIDATION_FAILED`、`KEY_NOT_ACTIVE`）。
- runtime 层：仅抛流程控制码（如 `UPLOAD_STAGE_TIMEOUT`、`TRACE_BROKEN`）。
- adapter 层：保留厂商原始码在 `adapterErrorCode`，同时映射平台码（如 `ADAPTER_NETWORK_ERROR`）。
- UI 层：禁止直接造码，只消费平台码并做 i18n 渲染。

### 17.2 WAF 更新策略（建议）
- 触发时机：`queue_completed`（队列上传完毕后批量更新）。
- 频率上限：最小更新间隔 `minUpdateIntervalMs = 30000`。
- 更新失败回滚：使用“上一版规则快照”立即回滚。
- 连续失败保护：连续 3 次失败后暂停自动更新，转手动恢复。

### 17.3 编号复用并发一致性（建议）
- 锁粒度：按类型前缀加锁（如 img/video/file 各自独立锁），不是全局锁。
- 抢占冲突退避：`baseDelayMs = 50`，`maxRetries = 5`，指数退避。
- 复用校验失败：自动换下一个候选号继续尝试，直到重试上限。
- 冷却复用：`reuseDelayMs = 900000`（15 分钟，已纳入高级设置）。

### 17.4 直接可实施默认值（建议）
- 统一由配置中心提供：
  - `upload.timeout.key_allocation_ms = 3000`
  - `upload.timeout.hook_ms = 60000`
  - `upload.timeout.adapter_ms = 120000`
  - `upload.threshold.slow_ratio = 0.5`
  - `upload.progress.report_interval_ms = 1000`
  - `upload.progress.heartbeat_timeout_ms = 15000`
  - `waf.update.min_interval_ms = 30000`
